Detects inconsistencies between the `@Disabled` annotation and the `@DisplayName` prefix in JUnit tests.

This inspection ensures that JUnit test methods annotated with `@Test` and `@DisplayName` follow a specific naming convention based on their enabled or disabled state. If a test is marked with `@Disabled`, its display name should start with a configurable prefix (defined in the settings). Conversely, if a test lacks the `@Disabled` annotation, its display name should not begin with that prefix. Violations are highlighted to maintain consistent test naming practices.

See official documentation for more details on writing inspections.

Example of a problematic test method: 


@Test
@DisplayName("DISABLED: Should process data")
public void testMethod() {
    // Test code here
}

Customize the expected prefix for disabled tests via inspection settings.
